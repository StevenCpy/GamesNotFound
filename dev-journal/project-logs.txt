What I learned today:
- A JWT token helps reduce database lookups to verify user.  It contains a signature and payload, which are enough.

What I learned today:
- List can be converted to set if array is one-dimensional eg. [{ID: 1, name: One}, {ID: 2, name: Two}] can be converted
to [1,2] using a map, then this can become a set (1,2).
- List can be converted to hash map eg. [{ID: 1, name: One}, {ID: 2, name: Two}] can be converted to
[[1, {ID: 1, name: One}], [2, {ID: 2, name: Two}]] which becomes a list of key-value pairs with ID as key.

- Error 405: "Method Not Allowed" caused when for eg. sending GET request to a POST endpoint.

What I learned today:
- Accessing a value in a hashmap (created using new Map()) is not same as in Python.  For eg. hashMap[2] doesn't work while hashMap.get(2) does.
- "select" after "insert" in Supabase SQL queries allows selecting and returning particular columns from the inserted data.
Eg. insert({"username": "Bill", "gameID": 1}).select("gameID") returns only the gameID column of the inserted row.
- Treat objects in state as immutable, always replace instead of mutating them
To change a list state, create a new list containing the previous list and add the new element.  Don't add to the original list itself.
Eg. setLibraryList([...libraryList, entry]) instead of libraryList.push(entry); setLibraryList(libraryList).
For sets, setLibrarySet(new Set(librarySet).add(entry)) instead of setLibrarySet(librarySet.add(entry)).

DAY 16
- Implemented "+ Add to Library" feature on Store

What I learned today:
- To remove an item from array state, create a new list with a filter for eg. setLibraryList(libraryList.filter(game => game.gameID != idToRemove))
- map has an index for eg. array.map((element,index) => {})
- For sets, librarySet.add() returns a set while librarySet.delete() returns a boolean.
- Comma operators evaluate the operands and return the last operand.  For eg. return (librarySet.delete(entry), librarySet) returns librarySet without entry.
